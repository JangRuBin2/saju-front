1 프로젝트 현황 진단 (Current State Analysis)

현재 Epic Fortune 프로젝트는 기술적인 기반(Next.js, FastAPI, Streaming AI)은 매우 탄탄하나, 상품성(Product Market Fit)과 수익화 모델(Monetization) 측면에서 개선이 시급합니다.

1-1. 완성도 강점 분석 (5가지)

  • 기술 스택 우수성: Next.js + FastAPI + Redis 아키텍처는 확장성과 성능 면에서 매우 우수함.
  • AI 통합: Server-Sent Events(SSE)를 통한 실시간 스트리밍 구현은 사용자 경험에 큰 강점.
  • 글로벌 준비: 4개국어 지원 및 다국어 아키텍처가 이미 구현되어 있음.
  • 디자인 시스템: "Midnight Bookshop"이라는 명확한 브랜드 컨셉 보유.
  • 결제 인프라: TossPayments 기본 연동 완료로 즉시 수익화 가능 상태.

1-3. 경쟁 서비스 분석 및 차별화

기존 시장의 주요 플레이어들과의 비교 분석입니다.

  • 점신/운세나라 (전통 강자): 방대한 DB와 커뮤니티가 강점이나, UI가 올드하고 AI 해석이 아닌 룰 기반 텍스트 조합 방식.
    → Epic Fortune 전략: 압도적인 AI 자연어 해석 품질과 모던한 UI로 공략.
  • 헬로우봇 (챗봇형): 친근한 캐릭터가 강점이나 깊이 있는 분석보다는 엔터테인먼트 성격이 강함.
    → Epic Fortune 전략: "정통 명리학의 현대적 해석"이라는 전문성 강조.
  • AI 사주 앱들: 최근 우후죽순 생겨나고 있으나 대부분 GPT-3.5 수준의 얕은 해석.
    → Epic Fortune 전략: Claude 3 Opus/Sonnet을 활용한 초고품질 심층 분석.

2 가격 정책 전면 개편 (Pricing Strategy Overhaul) 긴급

2-1. 현행 가격의 문제점 상세 분석

현재 1,100원(~$0.80) 모델은 PG사 수수료(약 3.3% + 부가세)와 AI 토큰 비용을 제외하면 수익성이 거의 없습니다. 또한, '저렴한 운세'라는 인식을 주어 브랜드 가치를 하락시킵니다. '프리미엄 컨설팅' 모델로 전환해야 합니다.

원가 분석 예시: 1,100원 결제 시 → PG수수료 36원, AI 토큰비용(Sonnet) 약 150원, 서버비용 등 제외하면 마진이 800원 미만. 마케팅비(CAC)를 감당할 수 없음.

2-2. 신규 가격 정책 상세 제안

A. 무료 티어 재설계 (Free Tier)

  • 기존: 매일 1회 무료 (과도한 혜택)
  • 변경: 회원가입 시 "스타터 크레딧 3회" 지급 (평생 1회). 이후 무료 없음.
  • 제한 사항: 무료 체험 시에는 Claude Haiku 모델 사용 + 300자 핵심 요약만 제공. "상세 해석 잠금" UI 적용하여 결제 유도.

B. 크레딧 단건 구매 (Pay-As-You-Go)

서비스명 가격 특징
오늘의 운세 1,900원 하루 24시간 동안 유효
사주풀이 (기본) 3,900원 평생 소장, 기본 분석
사주풀이 (심층) 5,900원 Claude Opus 모델 사용, 심층 분석
궁합 분석 4,900원 두 사람의 조화도 분석
연간 운세 9,900원 1년 전체 월별 운세 (객단가 상승 견인)

C. 구독 플랜 (Subscription)

플랜명 가격 (월) 제공 혜택
Basic 9,900원 오늘의 운세 무제한 + 정밀 사주풀이 월 3회 + 월간운세 1회
Premium 19,900원 모든 서비스 무제한 + Claude Sonnet 풀 품질 + 우선 상담(AI)
Annual 연간 결제 월 구독료 × 10개월 (2개월 무료 할인 효과)

D. 크레딧 번들 (Volume Discount)

  • 5,000 크레딧: 4,500원 (10% 할인)
  • 10,000 크레딧: 8,000원 (20% 할인)
  • 30,000 크레딧: 21,000원 (30% 할인)

3 무료/유료 전환 전략 (Conversion Optimization)

3-1. 온보딩 플로우 설계

사용자가 처음 방문했을 때 즉시 가치를 느끼게 하는 것이 중요합니다.

  • Step 1 (유입): "당신의 운명을 읽어드립니다" 문구와 함께 생년월일 입력 유도 (로그인 없이).
  • Step 2 (맛보기): 만세력 차트와 기본적인 오행 분포만 보여줌 (AI 해석 없음).
  • Step 3 (유혹): "당신의 사주에는 '도화살'이 숨어있네요. 상세한 의미가 궁금하신가요?" 같은 티저 문구 표시.
  • Step 4 (전환): "결과 확인하기" 클릭 시 로그인 창 → "가입 축하 3회 무료 이용권 지급!" 모달 팝업.

3-3. 이탈 방지 팝업 (Exit Intent)

  • Trigger: 마우스 커서가 브라우저 상단(탭 영역)으로 이동할 때 감지.
  • Logic: `localStorage`를 사용하여 하루에 1번만 노출되도록 제한.
  • Content: "잠깐! 지금 가입하면 내일의 운세가 무료입니다." 또는 특별 할인 쿠폰 제시.
// hooks/useExitIntent.ts import { useEffect, useState } from 'react'; export const useExitIntent = () => { const [isVisible, setIsVisible] = useState(false); useEffect(() => { const handleMouseLeave = (e: MouseEvent) => { if (e.clientY <= 0) { // Top of the browser const lastShown = localStorage.getItem('exit_intent_shown'); const now = new Date().getTime(); // 하루(24시간)에 한 번만 표시 if (!lastShown || now - parseInt(lastShown) > 24 * 60 * 60 * 1000) { setIsVisible(true); localStorage.setItem('exit_intent_shown', now.toString()); } } }; document.addEventListener('mouseleave', handleMouseLeave); return () => document.removeEventListener('mouseleave', handleMouseLeave); }, []); return { isVisible, close: () => setIsVisible(false) }; };

  • 결제 이탈 방지: 결제창에서 뒤로가기 시 "정말 취소하시겠습니까? 특별 할인이 사라집니다." 경고.

4 UI/UX 전면 개편 계획 (UI/UX Enhancement)

4-1. 홈페이지 (Landing Page) 개편

  • Hero 섹션: 정적인 이미지 대신 Three.js나 Canvas API를 활용한 별자리/오행 인터랙티브 파티클 애니메이션 적용.
  • 소셜 프루프: "현재 1,203명이 운세를 보고 있습니다", "누적 사주 분석 5만 건 돌파" 등의 실시간 데이터 표시 (신뢰도 상승).
  • 서비스 그리드: 각 카드에 가격 표시 추가, "가장 인기" / "신규" 뱃지 적용, 마우스 호버 시 결과 미리보기 줌인 효과.

4-2. 사주 입력 폼 완전 재설계

기존의 단순 입력 폼을 "운명을 알아보는 여정"처럼 느껴지도록 Step Wizard UI로 변경합니다.
  • Step 1 (기본정보): 이름, 성별 (아이콘 선택)
  • Step 2 (생년월일): 직관적인 DatePicker, 양력/음력 탭 전환, 윤달 자동 계산 체크박스.
  • Step 3 (태어난 시간): 시간 선택 or "시간 모름" 체크박스 (시간 불명 시 삼주 분석으로 전환 안내).
  • 피드백: 입력 완료 즉시 해당 일주(日柱)의 동물 캐릭터(예: 푸른 호랑이)를 애니메이션으로 보여주며 기대감 고취.

4-3. 결과 페이지 대폭 개선 (핵심!)

텍스트 위주의 지루한 결과 페이지를 시각적 정보로 변환합니다.

  • 인터랙티브 차트: 사주 팔자 8글자를 클릭하면 해당 글자의 의미(오행, 십신, 지장간)가 Popover로 표시됨.
  • 오행 분포 시각화: Recharts의 PieChart 또는 RadarChart를 사용하여 목화토금수 분포를 시각화. 과다/결핍 오행 표시.
  • AI 해석 카드: 성격, 재물, 연애, 직업, 건강 등 각 카테고리를 아코디언 카드 UI로 분리.
  • 공유/저장 바: 스크롤 시 상단에 고정되는 Sticky Bar에 "저장", "공유", "다시 보기" 버튼 배치.

4-4. 대시보드 전면 개편

  • 오늘의 운세 요약: 점수 게이지 바(0~100점)와 날씨 아이콘(맑음, 흐림 등)으로 직관적 표시.
  • 행운 아이템: 오늘의 행운 컬러(색상 칩)와 행운 숫자 뱃지 표시.
  • 캘린더 위젯: 이번 달의 길일(손 없는 날), 주의해야 할 날을 달력에 마킹.
  • 구독 배너: 무료 사용자인 경우 "프리미엄으로 업그레이드하고 모든 운세 잠금 해제" 배너 노출.

4-5. 모바일 UX 개선

  • Bottom Navigation: 모바일에서는 홈/운세/저장/마이페이지 탭 바 하단 고정.
  • 터치 제스처: 결과 페이지의 섹션 간 스와이프 이동 지원.
  • PWA 지원: 홈 화면에 추가할 수 있도록 manifest.json 및 아이콘 설정.

5 AI 해석 품질 고도화 (AI Quality Enhancement)

5-1. 현행 AI 구조의 문제점 분석

  • Haiku와 Sonnet 혼용으로 인한 품질 불일치.
  • 프롬프트가 리딩 타입별로 충분히 세분화되지 않음.
  • 피드백 루프 부재로 품질 개선이 어려움.

5-2. 모델 사용 전략 개편

  • 무료 (체험): Claude 3 Haiku 사용. 빠르고 비용 효율적. 300자 내외의 핵심 요약만 제공.
  • 유료 (표준): Claude 3.5 Sonnet 사용. 논리적이고 깊이 있는 해석 제공.
  • 프리미엄 (심화): Claude 3.7 Sonnet 또는 Opus 사용 (선택적). 복잡한 신살이나 평생 운세 등 고관여 서비스에 적용.

5-3. 프롬프트 엔지니어링 고도화 가이드

각 리딩 타입에 맞는 전문적인 시스템 프롬프트를 적용합니다. 아래는 개선된 프롬프트 예시입니다.

사주풀이 - 개선된 시스템 프롬프트 (한국어)

당신은 30년 경력의 명리학 전문가입니다. 다음 규칙을 반드시 준수하여 사주를 분석하세요. [분석 원칙] 1. **일간(日干) 중심**: 모든 분석은 일간을 기준으로 합니다. 일간의 오행과 강약을 먼저 파악하세요. 2. **강약 판단**: 월령(月令) 및 전체 오행 분포를 기반으로 신강/신약을 판단하고, 이를 해석의 핵심 기준으로 삼으세요. 3. **십신(十神) 해석**: 재성(財星), 관성(官星), 인성(印星), 비겁(比劫), 식상(食傷)의 분포와 의미를 해석에 포함하세요. 4. **대운(大運) 언급**: 현재 대운 기준으로 인생 전체 흐름 중 지금이 어떤 시기인지 설명하세요. [표현 원칙] 5. **건설적 어조**: 두려움을 주는 표현을 절대 금지합니다. "~할 수 있으니 주의하세요" 형식을 사용하세요. 6. **실용적 조언**: 각 영역 말미에 반드시 실천 가능한 조언(행운 색상, 방위, 마음가짐)을 포함하세요. 7. **금지 사항**: 특정 날짜 예측, 의료/법률 조언, 극단적 부정 예측은 절대 하지 마세요. [출력 형식] - 반드시 아래 마크다운 구조를 따르세요: ## ✨ 당신의 사주 핵심 (1~2줄 요약) ## 🌿 성격과 기질 (일간 기반 성격 분석) ## 💼 직업과 재능 (십신 기반 직업 적성) ## 💰 재물 운 (재성 기반 금전 운) ## 💕 연애와 인연 (배우자궁 및 도화 분석) ## 🏥 건강 (취약 오행 기반 건강 주의 사항) ## 🌊 현재 대운 흐름 (현재 대운 기간 및 조언) ## 🍀 행운의 열쇠 (행운 색상, 방위, 숫자, 마음가짐)

궁합 분석 - 개선된 프롬프트

두 사람의 사주를 비교 분석합니다. 다음 사항을 반드시 포함하세요: 1. **일간 오행 관계**: 두 사람의 일간 오행이 상생인지 상극인지 먼저 파악하세요. 2. **합충형파해(合沖刑破害) 분석**: 지지 간의 합·충·형·파·해 관계를 분석하세요. 3. **균형 잡힌 시각**: 장점과 주의점을 균형 있게 제시하세요. 나쁜 궁합도 "이 부분을 개선하면 좋은 관계가 될 수 있다"는 방향으로 서술하세요. 4. **실용적 소통 조언**: 두 사람의 차이점을 기반으로 구체적인 소통 방법을 제안하세요. [출력 형식] ## 💑 궁합 총평 (점수: XX/100) ## 🌟 잘 맞는 점 ## ⚡ 주의할 점 ## 💬 소통 & 관계 유지 팁

5-4. AI 응답 품질 평가 시스템

  • 결과 하단에 "이 해석이 도움이 되었나요?" (👍/👎) 버튼 추가.
  • 부정 피드백 시 사유 선택 모달 (너무 일반적임, 맞지 않음, 이해하기 어려움 등).
  • 피드백 데이터를 수집하여 프롬프트 튜닝에 활용 (RLHF 유사 방식).
  • FeedbackRating 테이블 스키마 추가.

5-5. 캐싱 전략 최적화

  • 오늘의 운세: TTL 6시간 (같은 날 재방문 시 동일 결과 보장).
  • 사주풀이/궁합: TTL 7일 (생년월일이 같으면 결과도 같아야 함 - 결정론적 일관성).
  • Redis Key Naming: `reading:{type}:{userId}:{dateHash}`

5-6. 스트리밍 UX 개선

현재 전체 텍스트가 한 번에 스트리밍되는 방식을 개선하여, 섹션별로 헤더가 먼저 뜨고 내용이 채워지는 방식으로 변경합니다. 로딩 스켈레톤 UI를 적용하여 대기 시간을 시각적으로 최소화합니다.

6 콘텐츠 확장 계획 (Content Expansion)

6-1. 신규 유료 리딩 타입 8가지

  1. 연간 운세 (Yearly Fortune): 신년 운세 시즌(12월~2월) 매출 견인. 토정비결 + 사주 명리학 결합.
  2. 이름 작명/감명 (Naming): 성명학(수리, 음령오행) 알고리즘 추가. AI가 좋은 한자 추천.
  3. 택일 (Date Selection): 결혼, 이사, 계약 등 중요한 날짜 3개 추천.
  4. 전생/내생 체험: 엔터테인먼트 요소가 강한 스토리텔링형 콘텐츠.
  5. 직업/적성 특화: MBTI와 유사하게 사주 기반의 적성 검사 및 직업 추천.
  6. 건강 운세: 오행의 불균형에 따른 취약 장기 분석 및 식습관 가이드.
  7. 자녀 양육 가이드: 아이의 기질 분석 및 맞춤 훈육법.
  8. 펫 사주: 반려동물의 생일을 기반으로 한 성향 분석 (틈새 시장 공략).

6-2. 셀럽 데이터베이스 확장

  • 로드맵: 현 60명 → 200명으로 확장 (배우, 스포츠 스타, 유튜버, 역사적 인물).
  • SEO 페이지: `/ko/celebrity/bts-jungkook` 같은 형태의 정적 페이지 생성 (Next.js `generateStaticParams`).
  • 데이터 구조: JSON 파일로 관리하며 생년월일, 프로필 이미지, 그룹명 등 포함.
